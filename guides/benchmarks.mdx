xxhash is among the fastest hashing libraries available for Go. The numbers below were generated on Ubuntu 20.04 with an Intel Xeon Platinum 8252C CPU under Go 1.19.2.

## Throughput: purego vs assembly (Sum64)

| Input size | purego    | asm       |
|------------|-----------|-----------|
| 4 B        | 1.3 GB/s  | 1.2 GB/s  |
| 16 B       | 2.9 GB/s  | 3.5 GB/s  |
| 100 B      | 6.9 GB/s  | 8.1 GB/s  |
| 4 KB       | 11.7 GB/s | 16.7 GB/s |
| 10 MB      | 12.0 GB/s | 17.3 GB/s |

<Note>
At 4 B, the pure Go implementation is slightly faster than assembly due to function-call overhead in the assembly path. Assembly wins on inputs of 16 B and above.
</Note>

## Running benchmarks

```bash
# Pure Go implementation
benchstat <(go test -tags purego -benchtime 500ms -count 15 -bench 'Sum64$')

# Assembly implementation (amd64/arm64 only)
benchstat <(go test -benchtime 500ms -count 15 -bench 'Sum64$')
```

Install `benchstat` if you don't have it:

```bash
go install golang.org/x/perf/cmd/benchstat@latest
```

## Available benchmarks

The test suite includes four benchmarks across the standard input sizes (4 B, 16 B, 100 B, 4 KB, 10 MB):

| Benchmark             | What it measures                                |
|-----------------------|-------------------------------------------------|
| `BenchmarkSum64`      | One-shot hashing of a `[]byte`                  |
| `BenchmarkSum64String`| One-shot hashing of a `string`                  |
| `BenchmarkDigestBytes`| Streaming: `New` + `Write` + `Sum64` per op     |
| `BenchmarkDigestString`| Streaming: `New` + `WriteString` + `Sum64` per op |

Run all benchmarks:

```bash
go test -benchtime 500ms -count 5 -bench .
```

<Tip>
Use `-count 15` with `benchstat` for statistically stable results, especially when comparing the `purego` and assembly builds side by side.
</Tip>
